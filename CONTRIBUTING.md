## ğŸ“ Coding Standards

### General Guidelines

* Follow existing code style and conventions
* Prefer clarity and maintainability over cleverness
* Avoid duplication; abstract common logic
* Include meaningful variable and function names

### Documentation

* Comment complex logic briefly and precisely
* Keep documentation consistent with code
* Use Markdown or in-code docstrings as appropriate

### File Organization

Maintain a logical structure (e.g., `src/`, `tests/`, `docs/`, `assets/`).
Group related files together to improve discoverability.
